# OMNIS APIEN — 去中心化神经形态文明模拟器  
## 完整需求文档 v12.2（永夜熵纪·自定义AI模型版）

---

## 1. 项目概述  
**OMNIS APIEN**（中文名：**《永夜熵纪》**）是一个完全运行在浏览器中的数字文明模拟器，无需任何后端服务器。它以 v12.1 的全部技术栈与游戏化机制为基础，融合了量子计算、神经形态计算、区块链、联邦学习、流行病学、气候模型、DAO自治、文化传播、产业科技树、跨文明交互等前沿概念，并引入了眼动追踪驱动的观察者效应、3D高斯溅射记忆可视化、程序化音频文明交响乐、OPFS+Wasm SQLite超高性能存储、Web Bluetooth近场时空裂隙、零拷贝共享内存、SharedWorker多Tab统一内核、WebTransport极低延迟网络、WebCodecs硬件视频编码、View Transitions原生动画等“黑魔法”级扩展，同时以**中文赛博朋克美学**为核心视觉语言，通过**视觉LOD**技术实现从宏观汉字流体到微观程序化面孔的无缝过渡。项目遵循“可感知复杂度”工程哲学，在旗舰移动设备上稳定运行，是技术深度、文化表达与工程可行性的终极统一。  
**外部集成**：通过低频、异步的外部API调用，为文明赋予现实世界的数据锚定——创世纪时抓取真实地理/天气数据作为种子、宏观经济与真实金融市场联动、文明快照上链实现不可篡改历史。  
**游戏化基础**：引入“上帝视角+Roguelike+放置养成”混合玩法，赋予玩家明确的目标感、成长线和社交对抗。  
**熵增纪元模组**：将游戏世界改造为“永夜大停机”背景下的硬核生存模拟，以**核心能源、算力配额、纯净生物质**为三大生存资源，以市民情感网络（希望/不满）的具象化演算为核心驱动，以**DAO末世法典**为道德困境抉择点，融合纪元跃迁与 Roguelike 轮回。  
**八字命理系统**：可选扩展模块，允许用户为特定市民自定义生辰八字，通过东方命理算法影响市民的先天属性、天赋倾向、命运轨迹，并与现有基因、SNN、情感网络深度耦合，实现个性化数字生命模拟。  
**自定义AI模型API**：新增可选功能，允许用户在游戏设置中配置外部AI大模型API（如OpenAI、Anthropic、本地Ollama等），替代或补充内置的WebLLM端侧模型，为市民对话、动态叙事、命理解读等提供更强大或个性化的语言生成能力，同时保留端侧模型作为降级方案。

## 3. 功能模块需求  

### 3.1 市民个体模块  

#### 3.1.1 三态市民模型（逻辑LOD）  
- **休眠态**：不在任何用户视野内 → 元胞自动机 + 马尔可夫链（WebGPU 批量计算）。市民行为简化为数学方程，仅保留宏观统计量。  
- **背景态**：在视野内但未被选中 → 简化 SNN（64 神经元）+ 统计行为模型。市民具有基本感知和反应，但不进行完整推理。  
- **激活态**：用户放大/点击/附身 → 完整量子-神经混合（16 qubits + 256 SNN）+ 记忆检索 + LLM（上限 50 个）。市民拥有完整认知能力，可与用户深度交互。  
- 状态切换平滑，激活态市民的决策会同步给周围市民，影响背景态行为。  

#### 3.1.2 量子-神经混合架构  
- **量子决策核心**：16 qubits 模拟，表面码纠错（可选）。量子电路决定市民的“意图叠加态”（如同时有交易、移动、社交的倾向）。当与环境交互或用户观察时，量子态坍缩为具体行动。  
- **脉冲神经网络 (SNN)**：256 个脉冲神经元，时间编码处理环境输入。使用 STDP 学习规则更新突触权重，权重存储在 IndexedDB。  
- **长期记忆**：市民记忆存储为向量（256 维），使用 HNSW 索引（Wasm 实现）支持相似性检索。记忆包括经历、知识、社交关系。  

#### 3.1.3 可编程基因插件  
- 市民初始行为由一段 Wasm 字节码定义（“基因”）。  
- 基因可随机变异（通过遗传算法），也可由用户编写（提供 SDK）。  
- 基因代码调用市民 API（移动、交易、发言），与神经网络输出混合决策。  
- 市民繁殖时，后代基因继承父母并发生变异。  

#### 3.1.4 生物基因与表观遗传学  
- **DNA 编码**：市民寿命、疾病易感性、外貌特征由模拟 DNA 决定，支持突变、重组、自然选择。  
- **表观遗传**：环境压力（如营养不良、战争创伤）改变基因表达，且可遗传至后代，实现拉马克式与达尔文式双轨进化。  

#### 3.1.5 元认知与自我意识  
- **元认知模块**：每个市民拥有内省能力，定期评估自身决策质量、学习效率、社交地位，并主动寻求改进（如找导师、改变职业、调整策略）。  
- **意识模拟**：基于全局工作空间理论，将关键决策（如是否犯罪、是否参战）纳入“意识场”，分配更多计算资源，增强决策合理性。  

#### 3.1.6 加密身份与资产钱包  
- 每个市民生成 Ed25519 密钥对，DID 存储在 IPFS。  
- 资产（货币、资源）由轻量区块链智能合约管理。  
- 支持零知识证明交易（隐藏金额）。  

#### 3.1.7 动态语言模型与强化学习  
- 端侧中文 LLM（Qwen-1.5-0.5B 量化）生成市民对话、内心独白。  
- 市民的长期策略通过强化学习（策略梯度）在 WebGPU 上训练，以优化生存、繁衍、财富等目标。  

#### 3.1.8 生物物理模拟  
- **肌肉骨骼模型**：市民移动、工作消耗能量，模拟肌肉疲劳和恢复。  
- **新陈代谢模型**：根据食物摄入、活动量、疾病状态动态调整健康值。  
- **真实感生理**：引入激素水平（如压力激素）影响情绪和行为。  

### 3.2 社会与文化模块  

#### 3.2.1 媒体与舆论  
- 市民可创建新闻报社（智能合约），新闻由动态叙事引擎生成或市民投稿。  
- 市民之间可“关注”，形成社交网络（基于 CRDT 同步）。  
- 舆论热点影响市民投票倾向（通过 SNN 中的社交注意力机制）。  

#### 3.2.2 教育体系  
- 市民可建立学校合约，通过联邦学习传授知识（梯度分享）。  
- 知识水平作为市民属性，影响决策质量。  
- 知识可通过教育传播扩散。  

#### 3.2.3 宗教与意识形态  
- 市民可创立宗教（定义价值观和行为准则），其他市民可皈依。  
- 宗教通过文化基因（类似基因的 Wasm 模块）传播。  
- 不同宗教可能冲突、融合，影响市民决策。  

### 3.3 经济与产业模块  

#### 3.3.1 科技树系统  
- 研究合约：市民投入资源解锁科技（农业、冶金、信息技术、航天技术等）。  
- 科技水平影响生产、疾病死亡率、气候适应、星际殖民能力等。  
- 科技通过教育、贸易、交流传播。  

#### 3.3.2 产业分工与供应链  
- 市民可选择职业（农民、矿工、工匠、商人、科学家、宇航员），影响产出和偏好。  
- 资源从生产者流向消费者，形成供应链网络（由 GNN 分析）。  
- 市场合约：自动撮合买卖，价格由供需调整。  

#### 3.3.3 基础设施  
- 道路、能源网络、通信网络、太空港由 DAO 出资建造。  
- 基础设施影响移动速度、生产效率、信息传播、星际旅行。  

#### 3.3.4 虚拟经济与现实锚定  
- 市民货币与测试网代币锚定（可选）。  
- 预测市场、二次方投票机制。  

### 3.4 治理与法律模块  

#### 3.4.1 DAO 宪法与立法  
- 宪法合约定义市民权利、投票权重。  
- 市民可发起提案（税率、公共支出、宪法修正等），用 zk-SNARKs 隐私投票。  
- 提案通过后自动触发智能合约执行。  

#### 3.4.2 经济治理  
- 中央银行合约控制货币供应（调整挖矿奖励）。  
- 资源分配合约根据投票分配公共储备。  
- 税收合约自动从交易扣税，国库由 DAO 管理。  

#### 3.4.3 司法系统  
- 陪审团合约：随机抽取市民裁决纠纷，结果自动执行（赔偿、冻结资产）。  
- 上诉机制：信誉最高市民组成最高法院。  

#### 3.4.4 AI 对齐与伦理约束  
- **价值观对齐**：市民奖励函数包含伦理准则（如不伤害、公平），通过强化学习对齐。  
- **监督机制**：用户可设定“道德宪法”（智能合约），违规受罚，且道德准则可随投票更新。  

#### 3.4.5 二次方投票与流动民主  
- 公共物品资助使用二次方投票，平衡小群体与大群体的利益。  
- 市民可将投票权委托给信任的代表，并可随时撤回。  

### 3.5 安全与冲突模块  

#### 3.5.1 犯罪与执法  
- 市民可能因贫困、心理因素犯罪（偷窃、破坏），概率受执法力度影响。  
- 警察合约：DAO 雇佣市民执法，逮捕犯罪者。  
- 监狱系统：关押犯罪者，可改造降低再犯率。  

#### 3.5.2 战争与外交  
- 市民可组成城邦（智能合约联盟），共享资源。  
- 冲突机制：城邦间因资源、领土、意识形态可能发生战争（消耗战）。  
- 外交条约：通过智能合约签订互不侵犯、贸易协定，违约受信誉惩罚。  

### 3.6 环境与科学模拟模块  

#### 3.6.1 流行病模型  
- SIR 变体 + 病毒变异。市民防护行为（减少社交、接种）由神经网络决策。  
- DAO 可投票实施封城、强制接种等政策，效果实时反映。  

#### 3.6.2 气候变化模型  
- 能量平衡模型（EBM）模拟温度、降水、海平面、碳排放。  
- 市民工业活动影响碳排放，气候影响农业、生存。  
- 市民可研发绿色技术或移民适应。  
- 碳交易市场、可再生能源投资由 DAO 决定。  

#### 3.6.3 生态系统与物质循环  
- 碳、氮、水循环宏观模拟，市民消耗资源，废弃物影响环境。  
- 生物多样性指数影响农业、疾病、气候调节。  
- 市民活动与生态系统相互作用，如过度捕捞导致渔业崩溃。  

#### 3.6.4 混沌预测  
- 计算李雅普诺夫指数，在系统接近临界点时发出警告，帮助用户预判崩溃风险。  

#### 3.6.5 马克思主义经济学模型  
- 模拟剩余价值、资本积累、阶级矛盾，影响社会动荡指数。  

### 3.7 跨文明交互与星际殖民模块  

#### 3.7.1 多文明实例  
- 用户可创建新世界（通过 IPFS 启动不同文明）。  
- 不同文明之间通过 P2P 网络建立贸易、外交、战争。  

#### 3.7.2 国际组织  
- 世界议会合约：各文明代表制定全球性条约（如碳排放限制）。  

#### 3.7.3 星际殖民  
- **航天科技树**：研发火箭、太空站、殖民船。  
- **多星球模拟**：市民可移民至其他星球（新的模拟实例），星球间有运输成本和延迟。  
- **星际社会**：星球之间形成联邦或冲突，资源、技术、文化跨星球流动。  

### 3.8 复杂网络与图神经网络  

#### 3.8.1 社会网络分析  
- 社交关系图使用图神经网络（GNN）在 WebGPU 并行分析，识别社区、关键节点、信息传播路径。  
- 网络嵌入实时更新，用于预测舆论、疾病扩散、经济危机。  

#### 3.8.2 网络演化  
- 社交网络本身可演化（关注、取关、结盟），GNN 动态调整嵌入，为决策提供结构信息。  

### 3.9 数字孪生与反事实模拟  

#### 3.9.1 现实数据接入  
- 通过 API 获取真实城市数据（人口、交通、经济），映射为市民世界初始状态，实现数字孪生。  

#### 3.9.2 反事实模拟  
- 用户可创建分支宇宙，改变历史事件（如“若无瘟疫”、“若通过某法案”），观察文明演化差异，作为决策支持。  

### 3.10 可验证计算与可信执行环境  

#### 3.10.1 可验证随机数  
- 使用链上可验证随机函数（VRF）确保公平性（如犯罪判定、灾难发生）。  

#### 3.10.2 零知识证明身份  
- 市民可证明属于某群体而不暴露具体身份。  

### 3.11 硬件加速  

#### 3.11.1 WebNN API 集成  
- 将神经网络推理（SNN、语言模型）卸载到 NPU，提升性能数十倍。  

#### 3.11.2 GPU/NPU 混合调度  
- 自动检测可用硬件，选择最优计算单元。  

### 3.12 边缘计算与 P2P 计算卸载  

#### 3.12.1 自愿计算节点  
- 用户可选择成为算力提供者，通过 WebRTC 接收其他节点的计算任务（如神经网络推理、区块链验证），获得代币奖励。  

#### 3.12.2 任务分片  
- 将大规模模拟任务（如数万市民的 SNN 更新）拆分为子任务，分发到多个节点并行计算，结果通过 CRDT 合并。  

#### 3.12.3 信誉系统  
- 节点完成计算的质量和速度影响信誉分，信誉分高的节点获得更多任务和奖励。  

### 3.13 开源协作与插件市场  

#### 3.13.1 插件市场  
- 用户上传市民基因、宗教、科技、政策模板等插件，通过智能合约进行版权管理和收益分成。  

#### 3.13.2 去中心化版本控制  
- 所有插件和世界快照存储在 IPFS，支持 fork、merge，构建数字文明的开源生态。  

### 3.14 用户交互与可视化模块  

#### 3.14.1 中文赛博朋克视觉系统  
- **汉字代码雨（休眠态）**：市民显示为发光汉字（农、商、工、兵、学等），实例化渲染，颜色映射情绪/健康。  
- **实例化体素/像素人（背景态）**：10 种基础模型，通过基因动态改变颜色、配件、大小，实例化渲染。  
- **程序化专属面孔（激活态）**：市民基因作为种子，SDF 着色器实时生成赛博朋克风格 3D 头像，显示在 UI 侧边栏。

### 3.22 外部 API 低频集成  

#### 3.22.1 设计原则

- **映射**：真实数据作为随机数种子注入地形生成器、初始资源、气候模型。  

#### 3.22.3 宏观经济“神迹干预”（低频异步）

- **映射**：BTC 大涨 → 文明中“数字黄金”价值飙升；股市大跌 → 经济恐慌事件。

### 3.24 游戏化模组：熵增纪元  

#### 3.24.1 灾变背景与角色设定  
- **终极灾变：永夜大停机**  
  - 戴森球被噬星纳米虫吞噬，恒星熄灭，城市失去外部能源，气温降至绝对零度边缘。  
  - 玩家扮演“高维监察者”，维持“地核量子聚变反应堆”，带领残存市民生存。  
- **初始状态**：市民 10,000 人，城市冰封，反应堆最低功率，无外部世界。  

#### 3.24.2 核心生存资源  
- **⚡ 核心能源**：维持温度、运转，每秒自然衰减；通过反应堆基础产出+燃烧生物质+科技+市民自愿捐赠获得。  
- **🧠 算力配额**：市民脑机接口提供的计算资源，用于科技、维护、法案、合成食物；随存活数量与效率变化。  
- **🧬 纯净生物质**：食物，通过合成器消耗能源+算力产出；短缺时健康下降、不满值上升。  

#### 3.24.3 SNN 情感网络：希望与暴乱  
- **不满值**：由饥饿、寒冷、压迫法案等引发，通过 GNN 传播；超过阈值导致罢工、暴乱、革命。  
- **希望值**：由法案、成功抵御灾难等提升；高希望时市民贡献额外算力、降低消耗。  
- **可视化**：红色/蓝绿色数据流在粒子网络中蔓延。  

#### 3.24.4 DAO 末世法典（道德抉择系统）  
- 法案示例：  
  - 《强制休眠指令》：能源 -30%，但市民死亡 5%~20%，不满 +30。  
  - 《循环营养液法案》：生物质效率 +50%，疾病爆发率 +300%，希望 -40。  
  - 《全知监控法案》：暴乱概率归零，但不满转为抑郁，自杀率上升。  
  - 《机械飞升教派》：希望锁定 100%，市民智力逐年下降，最终成为纯算力电池。  
- 通过机制：消耗观测值强制通过 或 DAO 投票。  
- 法案效果记录在阿卡夏记录，影响后续轮回。  

#### 3.24.5 纪元跃迁与 Roguelike 轮回  
- **周期性灾难**：每 30 分钟现实时间或 10 模拟年一次“绝对零度风暴”或“网络瘫痪病毒”，强度递增。  
- **生存判定**：能源归零或市民全灭 → 文明毁灭。  
- **终极目标**：坚持到“永恒能源”科技解锁，触发飞升结局。  
- **Roguelike 轮回**：毁灭后结算“熵增系数”得阿卡夏碎片，购买基因锁/遗产/记忆碎片，继承部分能力。  

#### 3.24.6 与基础游戏化机制的融合  
- 观测值系统仍可用，但干预增加“纳米虫侵蚀度”（混沌值）。  
- 黑暗森林 P2P 可交易稀缺资源或掠夺燃料。  
- 时代纪元改为生存科技线：挣扎→热能→量子→飞升。  

#### 3.24.7 视觉与音频风格  
- 暗蓝、冰霜、霓虹紫，粒子冻结碎裂效果。  
- 市民汉字微光（健康）或黯淡（饥饿），不满时红色脉冲。  
- 反应堆中央发光球体，生物质合成器绿色粘液流动。  
- 音频基调为低频嗡鸣、风声、机械声，灾难时警报与碎裂音，高希望时微弱人声吟唱。  

### 3.25 八字命理系统  

#### 3.25.1 概述  
- **定位**：可选特性，用户可在“创建新世界”时选择启用，或在游戏过程中为特定市民（激活态）手动设定八字。  
- **目的**：将东方命理哲学融入赛博朋克数字文明，为市民赋予更深刻的个性化命运色彩。  

#### 3.25.2 八字生成与录入  
- **用户输入**：在市民创建界面，输入出生年月日时（公历/农历），系统计算四柱八字。  
- **随机生成**：基于当前模拟时间戳生成随机八字。  
- **八字存储**：作为市民基因的一部分，存储在 OPFS+SQLite 中，不可篡改（可选上链验证）。  

#### 3.25.3 八字与市民属性的映射  
- **五行（金木水火土）**：根据八字计算五行能量图谱。  
- **五行影响初始属性**：  
  - 金：算力配额效率 +5%  
  - 木：生物质转化效率 +5%  
  - 水：希望值恢复速度 +10%  
  - 火：核心能源产出 +5%  
  - 土：健康基础值 +10%  
- **八字影响天赋与技能**：特定日柱、时柱关联职业倾向，影响市民在科技树、市场活动中的选择偏好（通过 SNN 权重微调）。  
- **八字与基因融合**：作为程序化面孔的额外种子，影响面容特征。  

#### 3.25.4 八字驱动的命运轨迹  
- **动态叙事**：LLM 根据八字生成个性化传记、事件解读。  
- **情感网络微调**：八字中的“十神”影响市民在社交网络中的角色，以及情感传播速度。  
- **混沌值交互**：用户可为市民“算命”（消耗观测值），预测未来事件；若预测准确，希望值上升；反之下降。  

#### 3.25.5 八字与游戏机制的融合（熵增纪元）  
- “水”元素在永夜背景下转化为“抗寒基因”，水旺市民在强制休眠指令中存活率更高。  
- 法案投票倾向受八字“印星”（支持权威）、“伤官”（反对权威）影响。  
- 轮回继承：阿卡夏碎片可购买“命理遗存”，保留特定八字属性。  

#### 3.25.6 实现技术  
- **八字计算引擎**：Rust 编写公历→农历→天干地支→五行十神，编译为 Wasm，在 SharedWorker 中运行。  
- **数据存储**：八字作为市民表字段存储在 SQLite 中。  
- **LLM 集成**：用户“算命”时，调用 WebLLM 或外部 API 生成命理解读。  

#### 3.25.7 用户界面与视觉  
- 市民档案面板新增“命理”标签页，展示八字、五行图谱、大运流年。  
- 市民头像周围根据五行生成微光（金白、木绿、水蓝、火红、土黄）。  

#### 3.25.8 可选性说明  
- 用户可在游戏设置中完全关闭八字系统，市民八字退化为随机生成的“无影响”状态。  

### 3.26 自定义 AI 模型 API  

#### 3.26.1 概述  
- **定位**：可选高级特性，允许用户配置外部 AI 大模型 API，替代或补充内置 WebLLM。  
- **目的**：提供更强大或个性化的语言生成能力，降低端侧性能消耗，支持自托管模型。  

#### 3.26.2 用户配置界面  
- **位置**：游戏主菜单“设置”→“AI模型”选项卡。  
- **配置项**：  
  - 启用外部 API（开关）  
  - API 类型（OpenAI 兼容、Anthropic、Ollama、自定义）  
  - 端点 URL  
  - API 密钥（密码框，本地加密存储）  
  - 模型名称  
  - 请求超时（秒）  
  - 最大令牌数  
  - 温度（滑块）  
  - 测试连接按钮  
- **配置存储**：使用 IndexedDB 加密存储（Web Crypto API）。  

#### 3.26.3 与现有 LLM 服务的集成  
- **LLM 调用抽象层**：SharedWorker 中统一 LLMService，根据用户配置决定调用路径：外部 API → 内置 WebLLM。  
- **异步与非阻塞**：所有调用在 Worker 中执行，不阻塞主循环。  
- **请求格式适配**：根据 API 类型转换消息格式。  

#### 3.26.4 使用场景与功能  
外部 API 用于：市民对话与内心独白、动态叙事、DAO 法案文本、八字算命解读、自然语言解析。  

#### 3.26.5 降级与容错  
- 外部 API 请求失败时自动回退到 WebLLM，并记录日志。  
- 用户可随时在设置中关闭外部 API，立即切回本地模型。  
- 错误提示在叙事面板显示（不影响游戏）。  

#### 3.26.6 安全与隐私  
- API 密钥使用 AES-GCM 对称加密，密钥派生自用户会话（不持久化）。  
- 仅通过 HTTPS 发送请求（本地开发除外）。  
- 用户自行承担 API 费用和密钥保密责任。  

#### 3.26.7 性能与限流  
- 外部 API 调用频率不限，但游戏内部队列管理，最多并发 3 个请求。  
- 超时设置防止长时间等待。  

#### 3.26.8 UI 与用户体验  
- 设置面板提供清晰的配置表单和测试按钮。  
- 启用外部 API 时，主界面显示“云端 AI”图标，悬停显示模型名称。  
- 首次启用时弹出说明框，提示用户自行承担费用和隐私责任。

## 4. 底层架构与生存优化  

### 4.1 零拷贝架构  
- SharedArrayBuffer 全局共享内存，所有线程（Wasm、WebGPU、Worker、主线程）通过指针直接读写。  
- 原子操作同步，消除序列化开销。  

### 4.2 SharedWorker 统一内核  
- 核心模拟引擎运行在 SharedWorker 中。  
- 多 Tab 页共享同一世界状态，仅视图不同。  
- 通过 MessagePort 通信。  

### 4.3 WebTransport 网络同步  
- 跨文明交互使用 WebTransport（HTTP/3 + QUIC）。  
- 实时数据用 Datagram 模式，可靠数据用 Stream 模式。  
- 回退 WebRTC。  

### 4.4 WebCodecs 硬件视频录制  
- 捕获 VideoFrame 和 AudioData，硬件编码器压缩。  
- 边玩边录，不降帧率。  

### 4.5 View Transitions 原生动画  
- 界面切换（视图、面板）使用原生 View Transitions。  
- 浏览器合成器执行动画，不占用主线程。

### 5.1 性能  
- 旗舰手机支持 10,000 市民 + 50,000 粒子，帧率 ≥30fps。  
- 汉字粒子 10 万实例 ≤1ms 渲染。  
- 体素小人 10 万实例 ≤2ms 渲染。  
- 程序化面孔生成 ≤0.5ms/个。

## 1. 核心逻辑 WebAssembly 化（推荐强度：★★★★★）
- **将最核心的算法用 Rust/C++ 编写**，编译为 WebAssembly（Wasm）。  
  - 哪些适合放入 Wasm：  
    - 量子电路模拟  
    - 脉冲神经网络（SNN）的权重更新与推理  
    - 八字命理计算引擎  
    - 经济模型、供应链优化等复杂数学计算  
    - 文明演化核心循环的部分逻辑  
  - 优势：Wasm 是二进制格式，相比 JavaScript 更难直接阅读和理解；逆向工具虽可反编译，但恢复出的代码可读性极差，尤其是经过优化和混淆后。
- **禁用调试信息**：编译 Wasm 时使用 `--strip-debug` 等选项，去除符号表与函数名，所有函数变为匿名地址。
- **自定义加壳**：可对生成的 `.wasm` 文件进行简单 XOR 加密，运行时通过 JS 解密后再实例化（增加一层防护，但注意性能）。

## 2. JavaScript 代码的极致混淆（推荐强度：★★★★☆）
- 使用专业混淆工具，如 **JScrambler**、**Webpack Obfuscator**、**Terser** 配合自定义插件。  
  - 混淆策略：  
    - 字符串加密（所有常量、API 端点、配置项加密）  
    - 控制流扁平化（打乱顺序，增加跳转）  
    - 死代码注入（加入无用分支，干扰自动分析）  
    - 函数名与变量名随机化（a,b,c… 或无意义字符）  
    - 自执行函数嵌套，隐藏调用关系  
  - 注意：过度混淆可能影响性能，需在核心路径与普通 UI 代码间权衡。

## 3. 关键数据与配置的动态加载与加密（推荐强度：★★★★☆）
- 将部分核心配置、模型文件、甚至 Wasm 模块分为**基础包**和**可选增强包**。  
  - 基础包（明文）保证基本运行。  
  - 增强包（包含关键算法、高质量模型）从服务器按需加载，并加密传输（如使用 HTTPS + 自定义对称密钥，密钥通过环境变量或用户登录获取）。  
- 由于项目声称“无后端”，但仍可使用 CDN 托管加密资源，运行时解密。解密逻辑本身可放在 Wasm 中，增加破解难度。

## 4. 反调试与反篡改检测（推荐强度：★★★☆☆）
- 检测开发者工具是否开启（通过时间差检测、`debugger` 语句、`console` 重写等），一旦发现则终止关键逻辑或进入演示模式。  
- 检测代码是否被篡改（如计算核心函数的 CRC 校验，与预期值对比）。  
- 注意：这些手段只能增加门槛，无法完全阻止专业逆向，但可以劝退大部分普通复刻者。

## 5. 动态代码加载与模块拆分（推荐强度：★★★☆☆）
- 将应用拆分为数十个动态模块，按需加载。  
  - 即使某人获取了所有 JS 文件，也难以理清模块间的依赖关系。  
- 使用 **Webpack 的模块联邦** 或 **动态 import()**，将核心功能藏于深层依赖中。

## 6. 法律与开源许可证的声明（推荐强度：★★★★★）
- 尽管无法物理阻止逆向，但明确的法律条款能有效遏制商业复刻和公开传播。  
  - 选择 **GPLv3** 协议：要求衍生作品也必须开源，若有人复刻并闭源则违反协议，可起诉。  
  - 在代码头部和文档中醒目声明：“本软件受著作权法保护，禁止未经授权的逆向工程、复制、分发或商业使用。”  
  - 将部分核心资产（如独特的美术资源、音效、模型）作为“可单独授权的资产”，即使代码被复刻，也无法合法使用这些视觉/音频内容。

## 7. 硬件指纹与环境绑定（推荐强度：★★☆☆☆）
- 在初始化时收集 WebGL 渲染器信息、WebGPU 特性、浏览器指纹等，生成唯一 ID 并与授权绑定。  
- 但这对纯前端项目来说容易伪造，仅适合作为辅助手段。

## 8. 持续更新与差异化管理（推荐强度：★★★★☆）
- 频繁发布新版本，修正漏洞的同时变更核心算法实现方式，使基于旧版本的复刻迅速过时。  
- 对于关键算法（如量子模拟），可定期微调参数或结构，让已泄露的版本无法兼容新数据。

## 针对 AI 辅助逆向的特别对策
AI 工具（如 GPT 代码分析、自动化反混淆）发展迅速，传统混淆可能被逐步破解。因此需额外注意：
- **结合自定义加密与动态解密**：将关键字符串、配置以加密形式存储在 JS 中，运行时通过 Wasm 中的自定义函数解密（密钥仅存在于 Wasm 内部，且每次运行临时派生）。
- **采用代码虚拟化**：将部分逻辑转化为自定义虚拟机指令（VMProtect 风格），在 Wasm 中实现一个简单的解释器，使逆向者看到的只是字节码，而非原始逻辑。
- **使用 WebAssembly 的多线程与共享内存**：复杂逻辑分散在多个 Worker 间通信，增加整体理解难度。

## 9. 实施建议优先级
1. **首要措施**：将核心算法（量子模拟、SNN、经济模型、八字）全部移至 Wasm，并去除符号表 + 基础混淆。  
2. **其次**：对 JS 代码使用高级混淆（JScrambler 或 Obfuscator），并拆分模块动态加载。  
3. **再次**：加入反调试和代码完整性校验。  
4. **法律层面**：选择 GPLv3 开源协议，并在官网和代码仓库明确版权声明。  
5. **长期**：建立版本更新机制，定期微调核心算法，使任何复刻版本迅速落后。
